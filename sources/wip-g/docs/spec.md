# WIP G — 데일리 숫자 퍼즐 (가칭: 오늘의 셈)

WIP Labs(wiplabs.pages.dev)에 추가할 하루 한 문제 숫자 퍼즐. Digits/Countdown 계열 메커닉.
이 문서는 Claude Code에 그대로 투입 가능한 구현 스펙이다.

---

## 1. 제품 철학

- **하루 한 문제.** 난이도 선택 없음. "오늘 것 끝냈다"는 완결감이 핵심 재방문 동력.
- **세션 1~3분.** 짧게 자주. 광고 없음, 다크패턴 없음, 회원가입 없음.
- **전원 동일한 문제.** 같은 날 접속한 모든 사용자는 같은 문제를 푼다. 공유·비교가 성립하는 전제.
- **서버 제로.** 정적 호스팅(Cloudflare Pages)만으로 완결. 백엔드, DB, 크론 일절 없음.

## 2. 게임 규칙

### 문제 구성
- 숫자 6개 + 목표값 1개(3자리, 101~999).
- 숫자 구성: 작은 수 4개(1~10, 같은 수 최대 2회) + 큰 수 2개(15, 20, 25 중에서, 중복 불가).

### 플레이 규칙
- 숫자 두 개와 연산자(+, −, ×, ÷) 하나를 골라 결합하면, 두 숫자는 소모되고 결과가 새 타일로 풀에 추가된다.
- **중간 결과는 항상 양의 정수**여야 한다. 음수·0·분수가 되는 연산은 UI에서 차단(해당 연산 시도 시 짧은 흔들림 피드백).
- 모든 숫자를 쓸 필요는 없다. 목표값과 **정확히 일치**하는 타일을 만들면 클리어.
- 근접 인정 없음. 대신 **undo 무제한**(연산 1회씩 되돌리기) + **전체 리셋** 제공.
- 시도 횟수 제한 없음. 포기 버튼 없음(내일 다시 오면 새 문제).

### 별점
- 클리어 시 사용한 연산 횟수를 최적해와 비교:
  - 최적해와 동일: ★★★
  - 최적해 +1: ★★
  - 그 이상: ★
- 최적해(최소 연산 횟수)는 클라이언트 솔버가 계산(§4).

## 3. 문제 생성 — 결정적(deterministic) 시드

### 날짜 → 시드
- 기준 시간대는 **KST(UTC+9) 고정**. 기기 로컬 시간대 사용 금지.
- `dateStr = KST 기준 "YYYY-MM-DD"` → 문자열 해시(FNV-1a 32bit 등) → 시드.
- PRNG는 **mulberry32** 사용. 같은 시드 = 같은 난수 시퀀스 = 같은 문제. 이것이 "전원 동일 문제"의 담보 장치이며, 문제 DB는 존재하지 않는다.

### 역방향 생성 (풀 수 있음 보장)
1. 시드 PRNG로 숫자 6개를 규칙대로 뽑는다.
2. 뽑힌 숫자들로 무작위 연산을 3~5회 적용해 목표값을 도출한다(중간 결과 양의 정수 제약 준수).
3. 목표가 101~999 범위를 벗어나면 2를 재시도(같은 PRNG 시퀀스 이어서 사용 — 결정성 유지).

### 품질 검사 및 재시도
- 생성된 문제에 대해 솔버로 최소 연산 횟수를 계산한다.
- 통과 조건: **최소 연산 3회 이상** (2회 이하면 싱거움).
- 불통과 시 시드를 `hash(dateStr + "-1")`, `"-2"`, ... 로 바꿔 전체 재생성. 검사 로직이 결정적이므로 모든 기기가 동일한 재시도 경로를 밟아 최종 문제도 동일하다.
- 안전장치: 재시도 20회 초과 시 마지막 후보를 그대로 채택(이론상 도달 불가에 가깝지만 무한루프 방지).

### 알려진 트레이드오프
- 시드 로직이 클라이언트에 있으므로 코드를 열면 내일 문제를 미리 볼 수 있다. Wordle과 동일한 수준의 노출이며 감수한다.

## 4. 솔버

- 목적: 최소 연산 횟수 계산(별점 기준 + 생성 품질 검사).
- 방식: 숫자 멀티셋 상태에 대한 BFS/DFS + 메모이제이션. 상태 키는 정렬된 멀티셋 직렬화.
- 6개 숫자 기준 탐색 공간은 수십만~수백만 상태 수준으로 JS에서 1초 내 처리 가능. 단, **메인 스레드 블로킹 방지를 위해 Web Worker에서 실행**하고, 결과가 나오기 전에도 플레이는 가능해야 한다(별점 표시만 지연).
- 생성 시 품질 검사와 플레이 시 별점 계산에 동일 솔버를 재사용.

## 5. 상태 관리

- **localStorage 단일 키 `daily_puzzle_v1`**, JSON 직렬화:

```jsonc
{
  "schemaVersion": 1,
  "lastPlayedDate": "2026-07-14",   // KST 기준
  "todaySolved": true,
  "todayOps": 4,                     // 클리어 시 사용 연산 횟수
  "todayStars": 2,
  "streak": 7,                       // 연속 클리어 일수
  "maxStreak": 12,
  "totalSolved": 31,
  "history": {}                      // 확장 여지, v1에서는 미사용
}
```

- streak 규칙: KST 기준 어제 클리어 → 오늘 클리어 시 +1. 하루라도 빠지면 오늘 클리어 시 1로 리셋. 판정은 날짜 문자열 비교로만 수행(타임스탬프 계산 금지 — 시차 버그 방지).
- 진행 중 상태(현재 타일, undo 스택)는 메모리에만 두고 저장하지 않는다. 새로고침 시 오늘 문제를 처음부터. (클리어 여부만 영속.)
- 이미 클리어한 날 재방문: 문제 대신 결과 화면(별점, streak, 공유 버튼, "내일 KST 자정에 새 문제" + 카운트다운).

### 자정 롤오버
- 플레이 도중 KST 자정이 지나도 현재 세션의 문제는 유지한다. 클리어 판정·저장은 **문제를 로드한 날짜** 기준. 다음 로드 시 새 문제.

## 6. 공유

- 클리어 시(및 결과 화면에서) "결과 복사" 버튼 → 클립보드에 텍스트 복사:

```
오늘의 셈 #142 ★★☆
연산 4회 (최적 3회)
🔢➕➖✖️➡️🎯
wiplabs.pages.dev/wip-3
```

- `#N`은 서비스 기준일(에포크, 상수로 하드코딩)로부터의 경과 일수. KST 기준.
- 정답 수식이나 숫자는 공유 텍스트에 절대 포함하지 않는다(스포일러 방지).
- `navigator.clipboard` 실패 시 텍스트를 선택 가능한 상태로 표시하는 폴백.

## 7. UI/UX

- **모바일 우선**, 세로 한 화면에 전부 들어올 것(스크롤 없음).
- 레이아웃(위→아래): 목표값(크게) / 숫자 타일 그리드 / 연산자 4개 / undo·리셋 / 연산 히스토리(수식 목록).
- 인터랙션: 숫자 탭 → 연산자 탭 → 숫자 탭 = 결합. 선택 상태는 명확한 하이라이트. 잘못된 연산(비정수·음수)은 실행 대신 흔들림 + 선택 해제.
- 목표 달성 순간: 간결한 성공 연출(과한 파티클 금지) → 결과 카드.
- 사이트 톤 준수: WIP Labs 기존 팔레트(배경 `#f6f3ec` 계열), 라이트 모드 전용, 시스템 폰트 스택. 첫 방문 시 3줄 이내의 규칙 안내 오버레이(다시 보지 않음 체크).

## 8. 기술 스택

- `wiplabs.pages.dev/wip-3/` 경로에 배포되는 정적 페이지.
- **Vite + TypeScript, 프레임워크 없음(vanilla)**. 이 규모에 React는 과잉. 단일 페이지, CSR only.
- 외부 런타임 의존성 0. 빌드 도구 외 npm 패키지 금지(mulberry32, FNV-1a는 직접 구현 — 각 10줄 이내).
- 파일 구조 제안:
  - `src/seed.ts` — 날짜→시드, PRNG
  - `src/generator.ts` — 문제 생성 + 품질 검사 루프
  - `src/solver.ts` — 최소 연산 솔버 (Web Worker 진입점 포함)
  - `src/game.ts` — 게임 상태, undo 스택
  - `src/storage.ts` — localStorage 입출력, streak 판정
  - `src/ui.ts` — DOM 렌더링, 인터랙션
- 테스트: 시드 결정성(같은 날짜 → 같은 문제), 생성기 제약(범위·정수성), 솔버 정확성(알려진 케이스), streak 판정 — vitest로 최소 커버.

## 10. 검수 체크리스트

- [ ] 서로 다른 기기·브라우저에서 같은 날 같은 문제가 나오는가
- [ ] KST 23:59 → 00:00 넘어갈 때 진행 중 세션이 깨지지 않는가
- [ ] 생성된 문제 100일치 샘플에서 전부 풀 수 있고 최소 연산 ≥3인가 (스크립트로 검증)
- [ ] 솔버 실행 중에도 첫 인터랙션이 즉시 반응하는가 (Worker 분리 확인)
- [ ] localStorage 없는 환경(시크릿 모드 일부)에서 크래시 없이 플레이만 되는가
- [ ] 공유 텍스트에 정답 유추 가능한 정보가 없는가
