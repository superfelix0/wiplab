# Bear Market Risk Dashboard data

The F7 page reads `docs/data/bear-market-risk.json`.

The file is generated by `scripts/update_bear_market_risk.py`. The script reuses existing WIP Labs data:

- `docs/data/kospi-sentiment.csv`
- `docs/data/kospi-sentiment-meta.json`
- `docs/data/us-liquidity.json`
- `docs/data/ai-earnings.json`
- `docs/data/market-per.json`

Update workflow:

1. Run `python scripts/update_bear_market_risk.py`.
2. Review `summary.totalScore`, `lastUpdated`, `summary.interpretation`, and each generated indicator.
3. If an input data source is updated manually, rerun the script rather than hand-editing scores.
4. Add one row to `history` only when doing a manual override.

Manual fields, if needed:

1. Edit each item in `indicators`.
   - `score`: 0 to 2 points.
   - `statusKo` / `statusEn`: visible status label.
   - `observation`, `recentChange`, `judgment`, `data`, `criteria`, `interpretation`: card and detail copy.
   - `sources`: keep empty or TODO until a real source is actually used. When connected, add source name, institution/type, URL, checked date, and notes.
2. Keep `sourceFramework.video` fields verified. Do not invent missing fields.

Score bands:

- 0-2: Normal
- 2.5-4: Watch
- 4.5-6: Caution
- 6.5-8: Alert
- 8.5-10: Risk

Current state:

- `sample: false` means the dashboard is calculated from connected data/proxies.
- The IPO indicator is still marked `dataPending: true` until a live IPO source is selected.
