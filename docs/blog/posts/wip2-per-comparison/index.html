WIP 2의 Forward PER 그래프를 막대그래프로 바꾼 이유

처음에는 Forward PER를 선형 그래프로 연결했지만, 이 값들은 동일한 날짜축 위의 시계열이 아니었습니다. 역사 평균, 최근 기사 기준, 현재 표시값처럼 서로 성격이 다른 기준점을 순서대로 놓은 것이었기 때문에 선으로 연결하면 실제 추세처럼 오해될 수 있었습니다.

그래서 각 지수별로 역사 평균과 현재값만 묶어 막대그래프로 나란히 비교하도록 바꿨습니다. 데이터의 한계를 숨기기보다 화면 구조에서 드러내는 편이 이 서비스의 방향에 더 맞습니다.

다음 단계에서는 각 지수의 Forward PER 원천 데이터를 더 안정적으로 확보해 실제 시계열 차트로 확장할 수 있는지 검토할 예정입니다.
